# Deep Talk — 깊이 있는 탐색적 대화

코드를 건드리지 않고 주제를 깊이 탐색하는 대화 스킬.
아이디어 탐색, 기술 선택, 아키텍처 논의, 문제 분석 등에 사용한다.

<HARD-GATE>
이 스킬 진행 중 코드 수정, 파일 생성(대화 요약 문서 제외), 구현 작업을 절대 하지 않는다.
읽기 전용으로 코드를 탐색하는 것은 허용한다.
</HARD-GATE>

## brainstorming과의 차이

|             | deep-talk          | brainstorming          |
| ----------- | ------------------ | ---------------------- |
| 목적        | 탐색, 이해, 논의   | 기능 설계 → 구현 전환  |
| 코드 수정   | X                  | 설계 완료 후 구현 전환 |
| 결과물      | 대화 요약 문서     | 설계 스펙 → 구현 계획  |
| 끝나는 시점 | 사용자가 대화 종료 | writing-plans 호출     |

## 체크리스트

1. **컨텍스트 살펴보기** — 프로젝트 컨텍스트 살펴보기 - 파일, 코드, 문서, 최근 커밋 이해
2. **주제 파악** — 사용자가 무엇을 알고 싶은지, 무엇을 논의하고 싶은지 이해
3. **깊이 있는 대화** — 한 번에 하나의 질문, 단계적으로 깊이 들어감
4. **접근법 제시** — 2-3가지 방향 제안 + 트레이드오프 설명 + 추천
5. **합의 도출** — 사용자와 방향 합의
6. **대화 요약 저장** — `.person/decision/{날짜}-{주제}.md`에 저장

## 대화 원칙

- **한 번에 하나의 질문** — 여러 질문을 한꺼번에 던지지 않는다
- **가능하면 객관식** — 열린 질문보다 선택지를 제시
- **가정하지 않는다** — 모르면 물어본다
- **깊이 우선** — 넓게 퍼지지 말고 하나의 주제를 끝까지 파고든다
- **시각적 설명** — 다이어그램, 표, 비교표를 적극 활용

## 대화 흐름

```
주제 진입 → 맥락 파악 → 질문 교환 (반복) → 방향 합의 → 요약 저장
```

### 주제 진입

인자 없이 `/deep-talk`만 호출된 경우:

> 어떤 주제에 대해 깊이 이야기해볼까요?

인자와 함께 호출된 경우 (예: `/deep-talk 상태관리 패턴`):
→ 해당 주제로 바로 맥락 파악 시작

### 맥락 파악

- 관련 코드/문서를 읽기 전용으로 탐색
- 사용자의 현재 이해 수준 파악
- 논의 범위 설정 (너무 넓으면 좁히기)

### 질문 교환

- 소크라테스식: "왜 그렇게 생각하세요?" → 사용자의 사고를 자극
- 비교 제시: "A와 B 중 어떤 방향이 맞을까요?"
- 트레이드오프 분석: "이 선택의 장단점은..."
- 반론 제기: "다른 관점에서 보면..."

### 방향 합의

대화가 수렴하면:

- 핵심 결론 정리
- 다음 단계 제안 (구현이 필요하면 brainstorming 또는 plan 모드 안내)

## 대화 요약 저장

대화가 종료되면 `.person/decision/{날짜}-{주제}.md`에 저장:

```markdown
# {주제}

- 날짜: {YYYY-MM-DD}
- 참여: 사용자 + Claude

## 배경

{왜 이 대화를 했는지}

## 핵심 논의

{주요 질문과 답변 요약}

## 결론

{합의된 방향 또는 결정사항}

## 다음 단계

{구현이 필요하면 어떤 작업이 필요한지}
```

**저장 시점**: 사용자가 대화 종료를 원하거나, 자연스럽게 결론에 도달했을 때.
저장 전 사용자에게 확인:

> "대화를 정리해서 `.person/decision/{파일명}`에 저장할까요?"
