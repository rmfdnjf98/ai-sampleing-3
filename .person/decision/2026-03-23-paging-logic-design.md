# 사용자 중심 페이징 및 리다이렉션 설계

- 날짜: 2026-03-23
- 참여: 사용자 + Gemini CLI

## 배경

현재 게시판의 페이징 시스템은 0번 인덱스부터 시작하며, 잘못된 페이지 번호(음수 등)에 대한 방어 로직이 부족함. 이를 개선하여 사용자 친화적이고 견고한 시스템을 구축하고자 함.

## 핵심 논의 및 결론

1. **페이지 번호 체계**
   - **사용자**: 1, 2, 3... (직관적 순서)
   - **시스템**: 0, 1, 2... (DB Offset 계산용)
   - **변환**: Controller에서 `int requestPage = page - 1;` 로직 수행.

2. **유효성 검사 및 리다이렉션 (핵심 결정)**
   - 잘못된 주소 입력 시 내부 보정 대신 **브라우저 리다이렉션**을 수행하여 주소창의 신뢰성을 확보함.
   - **하한선**: `page < 1` 인 경우 `?page=1`로 리다이렉트.
   - **상한선**: `page > totalPages` 인 경우 마지막 페이지 번호로 리다이렉트.

3. **데이터 구조 및 성능**
   - 전체 페이지 수 계산을 위해 `BoardRepository.countAll()` 호출 필수화.
   - 단순 `List` 반환 대신, 페이징 메타데이터를 포함한 `PagingDTO`를 정의하여 화면에 전달.

## 결론

- **방식**: Controller 기반의 명시적 리다이렉션 및 유효성 검사.
- **장점**: 깨끗한 URL 관리, 사용자 혼란 방지, SEO 최적화.
